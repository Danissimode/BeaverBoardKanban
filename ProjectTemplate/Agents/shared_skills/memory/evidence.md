## Key patterns

- Attach evidence when moving to Review or Done
- Always fill `checks` (what was verified) and `risks` (trade-offs, affected call sites)
- Required for Done Gate to pass
- Valid types: summary, test-results, build-log, diff, screenshot, benchmark, security-scan, other
